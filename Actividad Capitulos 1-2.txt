4.
•	Los datos son información codificada que puede ser procesada y transmitida por sistemas informáticos. 
•	Una red de datos es un conjunto de dispositivos interconectados que pueden comunicarse entre sí para compartir recursos y datos. 
•	La arquitectura de red se refiere a la estructura y diseño de una red, incluyendo componentes, protocolos y topología. 
•	Una red convergente es una infraestructura de red que combina voz, datos y video en una sola red para optimizar la eficiencia y reducir costos.


5.
•	Tolerante a fallas: Capacidad de una red para mantener la conectividad y el funcionamiento incluso en caso de fallos. Ejemplo: Redes de malla en las que cada nodo tiene múltiples rutas de conexión para evitar puntos únicos de falla. 
•	Escalabilidad: Capacidad de la red para crecer y adaptarse fácilmente a nuevas demandas y usuarios. Ejemplo: Redes basadas en tecnologías como el protocolo IPv6 que permite un número mucho mayor de direcciones IP. 
•	Calidad del servicio: Garantiza que los diferentes tipos de tráfico tengan prioridades adecuadas para satisfacer los requisitos de las aplicaciones. Ejemplo: Priorización de tráfico de voz sobre IP (VoIP) para garantizar una conversación clara y sin interrupciones. 
•	Seguridad: Protección de la red y los datos contra accesos no autorizados y ataques maliciosos. Ejemplo: Implementación de firewalls y sistemas de detección de intrusiones para proteger la red contra amenazas externas.


6.
•	Las redes orientadas a la conexión establecen una conexión antes de transferir datos, garantizando la fiabilidad pero con mayor sobrecarga.
•	Las redes no orientadas a la conexión transmiten datos sin establecer una conexión previa, lo que reduce la sobrecarga pero puede ser menos fiable en entornos inestables.


7.
•	La calidad del servicio se refiere a la capacidad de una red para proporcionar diferentes niveles de servicio a diferentes tipos de tráfico.
•	Se necesita una buena gestión del ancho de banda, priorización de tráfico y control de congestión para mantener una buena calidad de servicio para aplicaciones que requieren baja latencia, como VoIP o videoconferencia.


8.
•	La calidad del servicio es crucial para garantizar un rendimiento óptimo de las aplicaciones y servicios en la red.
•	Afecta directamente la experiencia del usuario, especialmente en aplicaciones sensibles a la latencia o que requieren ancho de banda garantizado
